# PCA Interpretable Clustering
---
A two-stage analysis pipeline that classifies materials by chemical species and then applies Principal Component Analysis (PCA) to decompose the descriptor feature space into an interpretable latent representation — revealing how different material families cluster in property space and which descriptors drive the separation.

This sub-repository contains the full pipeline applied to **Debye temperature** ($\Theta_D$) prediction, using elastic and volumetric descriptors from the [Materials Project](https://materialsproject.org/) and [AFLOW](http://aflow.org/) databases.

## Overview

The pipeline is split into two sequential notebooks:

1. **`pca_data_classifier.ipynb`** — Parses chemical formulas from the raw dataset and assigns each material to a chemistry-based class (e.g., Oxide, Carbide, Halide). Outputs one Excel file per class into a `classified_data/` folder.

2. **`PCA_interpretable_clusstering.ipynb`** — Loads the classified Excel files, performs standardised PCA, and generates a suite of publication-quality figures and a structured Excel results workbook, together with an interactive HTML scatter plot for cluster exploration.

The target property throughout is the **Debye temperature** ($\Theta_D$, in K), and the four input descriptors are shear modulus ($G_{VRH}$), Young's modulus ($E$), volume per atom ($V_\mathrm{atom}$), and mass density ($\rho$).

## Input Files

### `PCA_data_unclassified.csv`

The raw dataset prior to chemical classification, sourced from the Materials Project and AFLOW databases. Each row is one material entry.

| Column | Symbol | Description |
|---|---|---|
| `material_id` | — | Unique material identifier |
| `formula` | — | Chemical formula (used for classification) |
| `VPA` | $V_\mathrm{atom}$ | Volume per atom (Å³/atom) |
| `D` | $\rho$ | Mass density (g/cm³) |
| `SM` | $G_{VRH}$ | Shear Modulus, Voigt–Reuss–Hill (GPa) |
| `YM` | $E$ | Young's Modulus, Voigt–Reuss–Hill (GPa) |
| `DT` | $\Theta_D$ | Debye Temperature — target property (K) |

### `symbol_conversion.txt`

A plain-text mapping file converting raw column names to LaTeX symbols used in all figures. Format is one assignment per line:

```
SM  = $G_{\mathrm{VRH}}$
YM  = $E$
VPA = $V_{\mathrm{atom}}$
D   = $\rho$
DT  = $\Theta_D$
# Lines starting with # are ignored
```

## Notebook Descriptions

### Stage 1 — `pca_data_classifier.ipynb`

Reads `PCA_data_unclassified.csv` (or any compatible Excel/CSV) and classifies each material by parsing its chemical formula using a rule-based priority system. Outputs one `.xlsx` file per chemical class into `classified_data/`.

**Classification logic** (evaluated in priority order):

| Priority | Class | Rule |
|---|---|---|
| 1 | Element | Single element only |
| 2 | Carbide | Metal/metalloid + C |
| 3 | Boride | Metal/metalloid + B |
| 4 | Nitride | Metal/metalloid + N |
| 5 | Oxide | Contains O |
| 6 | Halide | Contains any halogen (F, Cl, Br, I, At, Ts) |
| 7 | Metal-Metalloid | Metals only, or metals + metalloids, no true non-metals |
| 8 | Other | Everything else |

The parser handles standard formulas (`Fe2O3`), Hill-order (`C2H6O`), parenthesised groups (`Ca3(PO4)2`), and space-separated tokens in Materials Project format.

### Stage 2 — `PCA_interpretable_clusstering.ipynb`

Loads all `.xlsx` files from `classified_data/`, merges them with category labels, standardises the four feature columns (`SM`, `YM`, `VPA`, `D`), and runs scikit-learn PCA. Seven analysis modules then produce the full set of figures and tabular results.

## Analyses Performed

### 1. Explained Variance Decomposition
Scree plot of per-component and cumulative explained variance ratio, identifying how many PCs are needed to capture the descriptor variance.

### 2. PC Loading Biplot
Bar chart of descriptor loadings (eigenvector components) onto PC1 and PC2, showing which physical quantities contribute most to each principal axis and in which direction.

### 3. PC1 vs PC2 Scatter Map
Static scatter plot of all materials projected onto the PC1–PC2 plane, coloured and marked by chemical class. Reveals natural clustering and family separation in latent descriptor space.

### 4. Interactive Scatter Plot (HTML)
A Plotly-powered HTML file of the same scatter map, enabling hover-to-inspect (formula, $\Theta_D$, class) and interactive zoom/pan.

### 5. $\Theta_D$ Statistics vs PC1 / PC2
Binned mean ± standard deviation profiles of Debye temperature along PC1 and PC2 axes, with a 10th–90th percentile band — directly linking position in PCA space to target property value.

### 6. Material Count vs $\Theta_D$ Distribution
Histogram with smoothed KDE envelope of the $\Theta_D$ distribution, segmented by chemical class.

### 7. Property Contour Maps
Feature values ($G_{VRH}$, $E$, $V_\mathrm{atom}$, $\rho$, $\Theta_D$) interpolated and plotted as filled contour maps on the PC1–PC2 grid, giving a spatially resolved view of how each descriptor varies across latent space.

### 8. Property Threshold Analysis
Tabulates minimum and maximum descriptor values within the LOW and HIGH $\Theta_D$ regimes, quantifying the property ranges associated with extreme target values.

## Installation

**Step 1 — Clone the repository**

```bash

cd pca-interpretable-clustering
```

**Step 2 — Create a conda environment (recommended)**

```bash
conda create -n pca_env python=3.12 anaconda

```

**Step 3 — Install dependencies**

Run the provided setup script:

```bash
python setup_env.py
```

Or install directly from `requirements.txt`:

```bash
pip install -r requirements.txt
```

## Usage

The two notebooks must be run **in order**.

### Step 1 — Classify the data

1. Launch Jupyter:
   ```bash
   jupyter notebook pca_data_classifier.ipynb
   ```
2. Run all cells. The notebook prompts you to:
   - Provide the folder path containing `PCA_data_unclassified.csv`
   - Confirm or set the output folder for `classified_data/`
3. Eight `.xlsx` files will be written to `classified_data/`, one per chemical class.

### Step 2 — Run the PCA analysis

1. Launch Jupyter:
   ```bash
   jupyter notebook PCA_interpretable_clusstering.ipynb
   ```
2. Run all cells. The notebook prompts you to:
   - Provide the path to the `classified_data/` folder
   - Set the output folder (results are placed inside a `PCA/` sub-folder)
   - Optionally load `symbol_conversion.txt` for LaTeX axis labels
   - Confirm column-to-label assignments
   - Choose interactive scatter options (Plotly HTML output)
3. All PDF figures, the interactive HTML file, and the Excel workbook are saved to `results/PCA/` automatically.
